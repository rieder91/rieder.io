My Interests

Photography

In my spare time I do a lot of photography. I focus on landscape, macro and wildlife shots. This is also why a wide angle and telephoto lens are my best friends. When travelling I'm almost always taking my gear with me - although it is quite heavy. I'm currently using a Canon 7D with four lenses: wide angle (10-22mm), general purpose (17-55mm), macro (100mm) and telephoto (70-300mm). Given the limitations of a crop sensor I plan to eventually upgrade to full-frame. You can check out some of my shots on Flickr.

Security

I really enjoy analyzing code. Of course there are a lot of vulnerabilities with web applications (first and foremost injection attacks) but I am more passionate about overflow attacks. I find the low-level nature of buffer/heap overflows or return-to-libc attacks more pure. I also took part in several CTFs and the CCC is usually a must for me. Given my interest in security I try to not make my own code vulnerable - even tough it might take more time to complete. You can also find me on several sites offering hacking challenges like HackQuest or HackThisSite.

Programming

Like every computer science student I tend to code a lot. I've dabbled in C, Haskell, Java, Python, Bash, SQL, PHP, Javascript and Octave (Matlab). The ones I've used the most are definitely Java and Python. I'm always interested in learning new stuff and I find things like Node.JS fascinating - sadly I haven't had the time to take a closer look yet. Given Java's slow development I'm currently learning Scala, as I think it is the most promising successor for a JVM-based language. The blend of object oriented and functional language elements make it really attractive. I also wonder if I will actually live to see Lambda expressions in Java.

Java

Both at university and at work Java is often the language of choice. I've developed both client and server applications in Java using different frameworks like Eclipse RCP, JSF, Lucene, Hibernate or Spring. I prefer web based three-tier architectures with REST. Altough Java is often frowned upon, I actually think it is a good choice for many projects due to its simplicity, good library support and testability.

Automation

I get fed up with deploying applications and setting up development environments rather quickly. For this reason I use Vagrant with Puppet/Chef and container engines like Docker. Setting up a development database with Vagrant and distributing just the Vagrantfile to all the project members is so much easier than copying VMs or explaining everyone how to set up the database manually - which leads to people having different problems instead of just one common problem. I also think that for a successful agile development workflow, it is essential to use version control systems like Git and build servers.

Python

There are applications for which Java is not a good choice. Small scripts, complex algorithms or scientific use cases like A.I. or machine learning usually suffer from Java's verbose nature - which is why I use Python. Python's good readability and powerful nature make it an excellent choice for these problems. You can see some of my Python code here.
